# jetson-flash/

One-time, **host-side** provisioning of a fresh Jetson Orin Nano Super
Developer Kit with Jetson Linux (L4T 36.4.4 / JetPack 6.x, Ubuntu 22.04).

This sits at the very start of the Jetson lifecycle:

```
  bare board
      │
      ▼
  jetson-flash/   ← you are here (factory flash, ~once per device)
      │
      ▼
  jetson-agent/   ← the systemd daemon that runs on every boot
      │
      ▼
  OTA updates     ← see docs/ota-flow.md
```

Use this folder when:

- You just received a new Orin Nano and need to put L4T on it.
- You bricked one and need to reflash.
- You want to bake a custom rootfs (e.g. with extra packages
  pre-installed) before the agent ever runs.

For day-to-day work on a *flashed* Jetson, you don't need anything in here.

## Prerequisites

- A Linux host PC (Ubuntu 20.04 or 22.04 recommended). Won't work from
  macOS — `apply_binaries.sh` needs Linux loop-mounting.
- At least 50 GB of free disk space.
- A USB-C cable to connect the Jetson to your host PC.

## Step 1: Download and prepare the L4T environment

`setup_env.sh` automates the NVIDIA download dance:
fetch the L4T Driver Package + Ubuntu sample rootfs, extract them, and run
`apply_binaries.sh` to install NVIDIA's proprietary drivers into the rootfs.

```bash
cd jetson-flash
chmod +x setup_env.sh
./setup_env.sh
```

You will be prompted for your `sudo` password during the extraction phase.

After it completes, you'll have a `Linux_for_Tegra/` subdirectory here —
that's NVIDIA's flashing toolchain. **`Linux_for_Tegra/` is gitignored**;
it's regenerated by re-running `setup_env.sh`.

## Step 2: Create a default user

Before flashing, seed the rootfs with a default user so you can log in
after first boot.

```bash
cd jetson-flash/Linux_for_Tegra
sudo ./tools/l4t_create_default_user.sh -u bebop -p <password> -a -n jetson-nano
```

(Replace `<password>` and `jetson-nano` with whatever you want.)

## Step 3: Put the Jetson into Force Recovery Mode (FC RCM)

1. Power off the Jetson.
2. Connect it to the host PC over USB-C.
3. Place a jumper across the **FC REC** and **GND** pins on the button
   header.
4. Connect the power supply to power on the Jetson.
5. Remove the jumper.
6. Verify the host sees it:

   ```bash
   lsusb | grep NVIDIA
   ```

   You should see an "NVIDIA Corp." device.

## Step 4: Flash the Jetson

To flash the **Jetson Orin Nano Super Developer Kit** to an NVMe drive,
use the `l4t_initrd_flash.sh` tool from `Linux_for_Tegra/`:

```bash
cd jetson-flash/Linux_for_Tegra
sudo ./tools/kernel_flash/l4t_initrd_flash.sh \
    --external-device nvme0n1p1 \
    -c tools/kernel_flash/flash_l4t_t234_nvme.xml \
    -p "-c bootloader/generic/cfg/flash_t234_qspi.xml" \
    --showlogs \
    --network usb0 \
    jetson-orin-nano-devkit-super nvme0n1p1
```

### Command breakdown

| Flag                                                          | What it does                                                       |
|---------------------------------------------------------------|--------------------------------------------------------------------|
| `--external-device nvme0n1p1`                                 | Target rootfs on the NVMe drive.                                   |
| `-c .../flash_l4t_t234_nvme.xml`                              | Partition layout for the NVMe.                                     |
| `-p "-c .../flash_t234_qspi.xml"`                             | Bootloader config for the on-board QSPI memory.                    |
| `--showlogs`                                                  | Stream detailed flash logs.                                        |
| `--network usb0`                                              | Use the USB-network interface for the initrd flashing handshake.   |
| `jetson-orin-nano-devkit-super`                               | Board config for the Super Developer Kit.                          |
| `nvme0n1p1`                                                   | Target boot partition.                                             |

## Step 5: Boot up

After a successful flash the Jetson reboots automatically. Disconnect the
USB-C cable, attach a monitor + keyboard, and log in with the user from
Step 2.

From there, follow [`docs/onboarding.md`](../docs/onboarding.md) under
"Running the agent on a Jetson" to install `bebop-agent`.
